# Copilot Instructions for Project

This file provides configuration and instructions for GitHub Copilot usage in this project.

## General Guidelines
- Use clear, descriptive commit messages.
- Follow the project's code style and conventions.
- Write concise and meaningful comments.
- Prefer existing project patterns and libraries.

## Copilot Usage
- Use Copilot to suggest code, but always review and edit suggestions for correctness and security.
- Do not commit Copilot suggestions that include sensitive data or credentials.
- If Copilot generates code that is not understood, research or ask for clarification before using it.

## Prompt Engineering
- When writing prompts for Copilot, be specific about the desired outcome.
- Use comments to guide Copilot in generating code that fits the project context.

## Example Prompts
- "Create a REST controller for managing users."
- "Generate a JPA repository for the Product entity."
- "Write a unit test for the login service."

## Security and Privacy
- Do not use Copilot to generate or handle sensitive information.
- Review Copilot code for potential security vulnerabilities.

## Documentation
- Document any Copilot-generated code that is non-trivial or not self-explanatory.

---

For more information, see the [GitHub Copilot documentation](https://docs.github.com/en/copilot).
