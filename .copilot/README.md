# Como contribuir com sugestões do Copilot

1. Leia as instruções em `.copilot/instructions.md` para entender o contexto e os objetivos do projeto.
2. Utilize os prompts de exemplo em `.copilot/prompts.md` para orientar suas sugestões.
3. Siga os padrões e restrições definidos em `.copilot/patterns.md`.
4. Certifique-se de que todo código sugerido:
   - Utilize caminhos absolutos.
   - Implemente validação de entrada.
   - Não silencie exceções.
   - Siga a estrutura de pacotes e utilize records para DTOs.
5. Documente endpoints REST com exemplos de uso (curl) em Javadoc e markdown.
6. Adapte as sugestões conforme a evolução do projeto.

---

Estas orientações garantem que as sugestões do Copilot estejam alinhadas com as necessidades e padrões do projeto.
